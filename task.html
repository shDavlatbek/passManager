Task 01 Business Finance Manager Hard
Context
Small and medium businesses in Uzbekistan manage their company cash flow through WhatsApp voice messages, paper notebooks, or scattered Excel files. There is no single place where the finance team or business owner can see what's coming in, what's going out, and where the money is actually going. Build a real product that solves this — a Telegram bot the team can speak to for logging transactions, connected to a proper company finance dashboard where management can monitor, analyze, and control business finances.
Part 1 — Telegram Bot
  • The bot accepts both voice messages and text
    Voice must be transcribed and interpreted automatically The bot identifies the intent: logging income, logging an expense, requesting a report, asking a question about past data
  • Every transaction is captured completely
    Amount, type (income or expense), category, date Optional note or comment if the user adds one
  • The bot responds in a natural, clear way — not robotic
    Confirms what it recorded in a readable format If something is unclear, it asks a follow-up — it does not silently fail
  • The team can query company financial data through the bot
    "How much did we earn this week?" / "What did we spend on logistics this month?" The bot replies with real numbers from saved data
  • The user can correct or delete a transaction they just added
  • The system supports multiple categories for both income and expenses, and the user can create custom ones
  • The bot handles ambiguous or incomplete input intelligently
    If the user says something like "50,000 received today" without a category — the bot should decide how to handle it. Ask? Make a smart guess? That's your call. It should never silently save wrong data or crash on unclear input
Part 2 — Web Dashboard
  • A multi-page web application — not a single scrollable page
    Overview page with key metrics and recent activity Transactions page — full list with filters and search Analytics page — charts and breakdowns Categories management page
  • The Overview page shows what matters at a glance
    Total income, total expenses, and net for the current period Comparison with the previous period (e.g. this month vs last month) A quick-add form — the user should be able to log a transaction without opening Telegram
  • The Transactions page is fully functional
    Filter by date range, category, and type Inline edit and delete for any entry
  • The Analytics page should give the business owner a clear understanding of where their money is going and where it's coming from — how you present this is up to you
  • Data from the Telegram bot appears in the dashboard automatically — always in sync
  • The app has a proper onboarding experience for a brand new user
    What does someone see when they open this for the very first time with no data? Empty states should guide — not confuse
  • One feature we didn't ask for — build something you think this type of business genuinely needs. It will be evaluated.
⚠ UX Warning A dashboard that works but looks like a generic template will not pass review. We build products for real businesses. The interface should feel polished, intentional, and trustworthy. Pay attention to how information is organized, how data is presented, and how the layout guides the user — not just whether the numbers show up.

Any feature, detail, flow, or idea you add on top of what's written here will be rated separately and weighted heavily in our evaluation. The candidates who impress us most are always the ones who saw a problem we didn't mention and solved it anyway. Show us how you think when nobody is telling you what to build.
What We Leave Open to You

What language does the bot respond in? What default categories make sense for a small Uzbek business? How do you handle currency? How does the dashboard feel on first open — before any data exists? What is your one extra feature, and why did you choose it? These are your decisions. Make them deliberately.

Task 02 Password Manager Medium
Context
Companies deal with dozens of shared credentials — CRM logins, cloud platforms, social media accounts, internal tools, client portals. These passwords get passed around in Telegram chats, written in notes, or reused everywhere. Build a team password manager — a secure, well-structured web application where a company can store, manage, and control access to their shared credentials.
Part 1 — Vault & Core Features
  • A master password locks and unlocks the entire vault
    The vault is locked by default when the app loads How and when the session locks is your decision — think about what makes sense for real usage
  • All stored passwords must be encrypted — never readable as plain text anywhere in the system
  • Each saved credential contains: service or platform name, URL, username, password, and optional notes
    Credentials can be tagged or grouped by department, project, or category — how you structure this is your decision
  • A built-in password generator
    Configurable length Toggle uppercase letters, numbers, and symbols on or off One-click copy to clipboard
  • Every saved password displays a strength indicator — weak, fair, or strong
  • One-click copy for both username and password — without showing the password on screen if the user prefers
  • The app detects if the same password is reused across multiple entries and alerts the user
  • Data export — the user can download a backup of their vault
Part 2 — Interface & Experience
  • A multi-page application with clear structure
    Lock screen — this is the first thing the user sees every time Vault overview — all credentials in a searchable, filterable list Add / Edit credential form Settings page — change master password, manage export, customize preferences
  • The vault list is fast, searchable, and easy to navigate — how you organize and structure it is your decision
  • The add/edit form is clean and quick to use — every field should be obvious without any explanation
  • Smooth transitions between pages and states — the app should feel alive, not static
  • A proper first-time experience for a new user opening an empty vault
    Empty states should guide the user toward their first action — not show a blank screen
  • One feature we didn't ask for — build something you think a company managing shared credentials genuinely needs. It will be evaluated.
⚠ UX Warning A password manager that looks generic will feel untrustworthy — even if it works perfectly. The lock screen, the vault layout, the small interactions — all of it contributes to whether a team feels safe handing over their company credentials. This project will be judged as much on how it feels as on whether it works.

Task 03 AI Meeting Notes Hard
Context
Every business runs on meetings — and every meeting produces decisions, tasks, and commitments that get lost immediately after everyone leaves the room. Build a tool for business teams: a Telegram bot that turns meeting recordings into structured, actionable notes, connected to a company dashboard where the whole team can track decisions and follow up on action items.
Part 1 — Telegram Bot
  • The bot accepts a voice message or audio file sent in Telegram
    It should handle recordings of different lengths — not just short clips The bot sends a status update while processing — the user should never wonder if it's working
  • The audio is transcribed and analyzed by AI to produce structured output
    A short meeting summary — 3 to 5 sentences Key decisions that were made Action items — what needs to happen, and who is responsible (if names were mentioned) Main topics or agenda points that were discussed
  • The bot returns results in a clean, readable format — not a wall of text
    Clearly organized sections with labels Easy to read and forward to a colleague directly from Telegram
  • The user can also paste or send a text transcript directly — not everyone records audio
  • The user can ask the bot to adjust the output — "make it shorter", "more formal", "focus on action items only"
  • Every processed meeting is automatically saved and visible in the web dashboard
Part 2 — Web Dashboard
  • A multi-page application for managing all meeting notes
    Home — recent meetings and open action items at a glance Meetings list — all past meetings, searchable and filterable Meeting detail page — full notes, transcript, and action items for one meeting Action Items page — all tasks across all meetings in one place with status tracking
  • The Meetings list is functional and fast
    Keyword search across meeting content Filter by date range Each meeting card shows: date, summary preview, and number of open action items
  • The Meeting detail page is well-organized and editable
    Summary, decisions, action items, and topics in clearly separated sections Option to view or collapse the full transcript Any section can be edited manually if the AI output is wrong — how you design that editing experience is your call
  • The Action Items page keeps users on top of what needs to happen
    Items can be tracked by status — how you define and display those statuses is your decision If the same person is mentioned across multiple meetings, their tasks should be visible together
  • All data from the bot syncs to the dashboard automatically
  • A proper first-time experience — a new user with no meetings yet should know exactly what to do next
  • One feature we didn't ask for — build something you think makes this genuinely useful for a business team. It will be evaluated.
⚠ UX Warning Meeting notes are used by professionals every day. The dashboard should feel like a tool a team opens at the start of their workday — not a prototype. Clear hierarchy, clean layout, and a consistent visual language are not optional here. We will evaluate the interface as seriously as we evaluate the quality of the AI output.
